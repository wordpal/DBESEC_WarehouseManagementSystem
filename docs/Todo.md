### 🎯 调整后的四人本地开发分工

**核心目标：** 每人都在自己电脑上搭建环境，开发一个完整可用的本地系统。

#### **角色一：张三 (项目经理 & 数据库架构师)**

**你的终极任务：** 设计系统核心，并让团队每个人的本地数据库都能跑起来。

**你的具体任务清单：**

1.  **数据库设计与搭建 (必须最先完成！)**
    *   [ ] 最终确定《功能清单.xlsx》- 明确系统到底有哪些页面和功能。
    *   [ ] 绘制**ER图** (用Draw.io)，并填写《数据库表结构设计.xlsx》。
    *   [ ] 编写 `01_create_tables.sql` (包含建库`CREATE DATABASE IF NOT EXISTS warehouse;`、建表、建索引)。
    *   [ ] 编写 `02_sample_data.sql` (**插入基础支撑数据**，比如：2-3个仓库、5-10个物料分类、1-2个管理员账号。**先不插入大量的业务数据**)。

2.  **项目框架与核心模块**
    *   [ ] 创建Eclipse Web项目，配置好Tomcat。
    *   [ ] 创建 `DatabaseUtil.java` (数据库连接工具)。
    *   [ ] 创建所有Java Bean (如 `Product.java`, `Inventory.java`)。
    *   [ ] 创建核心DAO接口和实现，例如 `InventoryDAO.java` (包含基础的CRUD方法)。

3.  **团队支持**
    *   [ ] 将 `01_create_tables.sql` 和 `02_sample_data.sql` 发给所有组员，确保他们能成功初始化自己的本地数据库。
    *   [ ] 协助队友解决环境问题和数据库查询问题。

---

#### **角色二：李四 (后端功能开发工程师)**

**你的终极任务：** 实现所有让页面"动起来"的后台逻辑。

**你的具体任务清单：**

1.  **环境准备**
    *   [ ] 从张三那里获取SQL脚本，初始化自己的本地数据库。
    *   [ ] 从张三那里获取项目框架，导入Eclipse，确保能运行。

2.  **Servlet开发 (你的主战场)**
    *   [ ] `ProductServlet.java`: 处理物料的列表、新增、修改、删除。
    *   [ ] `InventoryServlet.java`: 处理库存查询、入库、出库操作。
    *   [ ] `ShipmentServlet.java`: 处理物流单据的创建和状态更新。
    *   [ ] `ReportServlet.java`: 处理统计查询，并实现**数据导出为Excel**的功能 (使用Apache POI库)。

3.  **高级功能 ("巧思"实现)**
    *   [ ] 在 `InventoryService.java` 中实现一个简单的 **智能货位推荐** (例如：随机推荐一个空闲货位)。
    *   [ ] 在 `ReportService.java` 中编写SQL，实现 **物流预警看板** 所需的数据查询 (查询状态为"延误"的运单)。

---

#### **角色三：王五 (前端页面开发工程师)**

**你的终极任务：** 制作所有用户能看到的页面，并实现流畅的交互。

**你的具体任务清单：**

1.  **UI组件库与基础页面**
    *   [ ] 在项目中引入Layui CSS/JS。
    *   [ ] 创建 `main.jsp` (主布局，含导航菜单)。
    *   [ ] 创建 `login.jsp` (登录页)。

2.  **业务页面开发 (每个页面都遵循相同模式)**
    *   [ ] **列表页** (如 `product_list.jsp`)：
        *   使用Layui Table组件。
        *   用AJAX调用李四的Servlet (如 `/product?action=list`) 来**填充表格数据**。
        *   实现 **"搜索"** 和 **"删除"** 按钮的功能。
    *   [ ] **表单页** (如 `product_form.jsp`)：
        *   使用Layui Form组件。
        *   实现 **"提交"** 按钮，通过AJAX将数据发送给李四的Servlet。
    *   [ ] 依次完成：物料管理、库存管理、物流管理等相关页面。

3.  **亮点功能实现**
    *   [ ] **首页看板 (`dashboard.jsp`)**：
        *   使用ECharts画一个简单的柱状图，显示各仓库库存量。
        *   画一个表格，显示"延误"的物流列表。
    *   [ ] **(可选) 多语言切换**：在页面右上角加一个下拉框，实现中英文标题的切换 (用前端JS字典)。

#### **角色四：赵六 (功能测试与数据填充员)**

**你的终极任务：** 保证系统质量，并通过添加丰富的数据让演示效果拉满。

**你的具体任务清单：**

1.  **环境与基础测试**
    *   [ ] 从张三那里获取SQL脚本和项目，搭建自己的本地环境。
    *   [ ] 按照《测试用例.xlsx》 (由你创建)，对张三和李四完成的每一个功能进行**暴力测试**，并记录Bug。

2.  **数据魔法师 (你的核心贡献)**
    *   [ ] **编写 `03_demo_data.sql`**：这是你最重要的任务！
        *   在这个文件里，编写INSERT语句，生成**大量、逼真、有关联**的演示数据。
        *   **例如**：插入50个不同的物料、插入100条入库记录 (让库存数量不为零)、插入30条物流单据 (其中5条状态设为"延误")。
        *   这些数据将直接决定系统演示时的专业度和观感。

3.  **端到端测试与文档**
    *   [ ] **系统流程测试**：扮演用户，从头到尾走一遍核心流程 (登录 -> 新增物料 -> 入库 -> 查询库存 -> 创建出库单 -> 发货 -> 查看报表)。
    *   [ ] **收集截图**：在测试过程中，对每个主要界面和操作结果进行**截图**。
    *   [ ] **编写《用户操作手册.docx》**：用一个简单的Word文档，用图文并茂的方式，说明如何运行和使用这个系统。
